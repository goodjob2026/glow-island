1. 图块精灵（Tile Sprites）

1.1 基础规格

项目 规格

颜色深度 RGBA 32-bit

1.3 四种状态规范

每个图块必须提供以下状态：

状态1：Normal（默认）
- 文件名：{tile_id}_{name}_normal@2x.png（如 t01_shell_normal@2x.png）
- 静态单帧 PNG，112×112px实际绘制区，四周8px透明出血
- 底部烘焙阴影：rgba(0,0,0,0.10) 偏移 0,4px（烘焙进图层）
- 纸纹叠加：6%不透明度，统一纹理文件

状态2：Highlighted（连接高亮）
- 文件名：{tile_id}_{name}_highlighted@2x.png
- 亮度 +20%，内发光 rgba(255,255,255,0.40)，scale效果由Runtime实现（不需要物理放大图层）
- 可通过Runtime Shader实现，此时只需提供Normal帧即可；若无Shader能力则绘制单独高亮帧

状态3：Disappearing（消除序列帧）
- 文件名：{tile_id}_{name}_disappear_{frame:02d}@2x.png（frame 01-12）
- 序列帧：12帧，30fps（总时长 0.4s）
- 帧1-3：图块scale 1.0→1.14（含透明出血区扩展），帧4-9：scale 1.14→0 + 旋转±12°，帧10-12：粒子散开（4-6个主题小图标）淡出至透明
- 粒子形状对应图块主题（贝壳块→贝壳碎片，樱花块→花瓣，鸟居块→红色碎片）
- 末帧全透明

状态4：Special（特殊块循环动画）
- 针对4种特殊块（河豚/金枪鱼/彩虹鱼/萤火提灯），提供Spine骨骼动画文件
- 循环动画：{type}_idle — 持续呼吸/脉冲动画（2.0-2.8s周期）
- 激活动画：{type}_activate — 触发时一次性播放（0.5-1.0s）
- Spine导出格式：.skel + .atlas + PNG纹理

2. 章节背景（Backgrounds）

2.1 基础规格

项目 规格
尺寸 1080 × 1920 px（Portrait，9:16基准）
安全区适配 内容主体在中央 1080×1600 区域内，顶底各留240px溢出
颜色深度 RGB 24-bit（无需透明通道）
导出格式 PNG-24 或 JPEG（Q95）
图层说明 分层导出（可选）：天空层 / 中景层 / 近景层，供视差滚动使用
风格 手绘水彩插画风格，AC式正视平面场景，非摄影拟真

2.2 每章两版本说明

  • 修复前（before）：整体低饱和度（HSL 饱和度 ≤ 15%），偏灰暗调，元素破败但轮廓完整（如铅笔速写稿）
  • 修复后（after）：满彩水彩质感，明亮温暖，参考各章节色板；场景中有角色和活跃的生活元素

2.3 章节背景描述

Chapter 1 — 海边码头（Beach Harbor）
- 构图：木码头从画面左下向右上延伸入海，右侧海面占画面40%，远处有小岛和灯塔剪影，近景左侧有系船木桩和贝壳风铃
- 修复前：灰白朽木码头，绳索松垂，海面暗淡无反射，阴天，画面整体HSL饱和度≤12%
- 修复后：木板暖棕#C4955A，海水青绿#5BBFB5，码头两侧悬挂橙色圆灯笼；光线：金色下午光从右上45°投射，海面有碎金光斑；小木船停靠右侧，船头有贝壳风铃摆动
- 棋盘内使用背景：浅蓝绿渐变底图（#A8DDD8→#C8EEE8），透明度25%，顶部有极淡海浪线装饰
- 特殊版本：夕阳版（bg_ch01_harbor_after_sunset@2x.png）— 天空橙红渐变#F4A261→#EE6A30→#1E2E50，海面金色光道，阿海在远处码头系绳剪影

Chapter 2 — 竹林村落（Bamboo Village）
- 构图：竹林小径从近景延伸至中景村落入口，近景两侧为翠竹，中景有竹制凉亭，背景为更深的竹林
- 修复前：竹子倒伏或枯黄，小路泥泞，凉亭屋顶破损，整体灰绿调
- 修复后：翠竹挺立#7DBE77，小径铺设浅沙色石板，凉亭有绿草顶，路边竹筒花器插有樱粉色山茶；光线：竹林间隙的午后光斑效果（圆形光斑散落地面）
- 棋盘内使用背景：竹叶绿渐变底图（#C8E8C0→#E0F0D8），透明度25%，有极淡竹节轮廓纹（透明度8%）
- 特殊版本：无（竹林场景光线变化不如港口戏剧性，无备用版本）

Chapter 3 — 樱花坡田（Sakura Hillside）
- 构图：石板坡道从近景向远处延伸，两侧交替种植粉白花田，中景有两棵盛开樱花树，远景可见海岸线
- 修复前：枯枝樱花树，花田荒芜如枯草，石板路碎裂，整体灰黄调，有淡薄雾
- 修复后：樱花盛开（粉白花簇#F0D8D8），花田行列整齐，石灯笼点缀坡道，清晨薄雾（白色低透明度层叠加于远景），阿花在中景捧着一束花；花瓣飘落粒子（静止帧绘制3-5片飘落花瓣）
- 棋盘内使用背景：樱粉渐变底图（#F8E8E8→#FFF5F5），透明度25%，边缘有散落花瓣轮廓

Chapter 4 — 松林神社（Pine Forest Shrine）
- 构图：石板参道正面透视，两侧石灯笼对称排列，中景为红色鸟居，背景为神社正面，松树形成天然框架
- 修复前：鸟居斜倒，石灯笼苔藓横生，参道石板缝草丛，神社门扉关闭，暗绿灰调
- 修复后：鸟居鲜红#C8402A直立，石灯笼内有暖黄灯光，参道石板清洁，注连绳新换，神社正面可见；光线：夜晚场景，灯笼光是主光源，树冠剪影形成天然顶棚，远处星空隐约可见
- 棋盘内使用背景：深林绿渐变底图（#3D5040→#5A7058），透明度20%，极淡松针剪影装饰（透明度8%）

Chapter 5 — 渔村港口（Fishing Village Port）

透明通道 RGBA，序列帧全部透明背景
风格要求 特效粒子形状对应自然元素（贝壳碎片、花瓣、气泡、竹叶），非抽象几何粒子

6.2 特效列表

EF01 — 图块消除特效（Tile Disappear）
- 触发：任意图块消除时
- 画布尺寸：160×160 @2x（含出血）
- 帧数：12帧 @30fps（总时长 0.4s）
- 内容：帧1-3图块放大1.14×；帧4-9缩至0同时旋转±12°；帧10-12：4-6个该图块的主题粒子向外散开淡出
- 主题粒子对应：T01贝壳→贝壳碎片；T03海浪→水滴；T05竹节→竹叶；T06石灯笼→光点；T09樱花→花瓣；T13松果→松针；T15鸟居→红色碎片；T17鱼→鱼鳞片；T19星光→星点
- 命名：fx_tile_disappear_{tile_id}_f{01-12}@2x.png（或使用颜色变体版：fx_tile_disappear_{color}_f{01-12}@2x.png，6色版）

EF02 — 连击气泡（Combo Popup）
- 触发：连击数 ≥ 3 时弹出
- 画布尺寸：220×110 @2x
- Spine文件（推荐）：fx_combo_popup.skel
- 内容：对话气泡形（底部有三角指向标）从触发位置弹入（spring曲线，scale 0→1.2→1.0，0.3s）→ 悬停1.2s → 上浮淡出（0.4s）
- 气泡样式：深珊瑚背景#C85A40，月光白边框3px，数字金黄#F5C842Bold，COMBO文字月光白Regular
- 变体：3×3/5×5/10×10连击字号递增（24px/32px/48px）

EF03 — 萤火光链效果（Firefly Chain）
- 触发：连接 Glow萤火提灯块 时
- 画布：全屏叠加层（1080×1920 @2x）
- Spine文件：fx_firefly_chain.skel，路径跟随连接路径动态调整
- 内容：金色光粒子（形如萤火虫，圆形光点+轻微尾迹）沿连接路径游动，速度约 500px/s，流向岛屿地图区域；途经图块时短暂发亮
- 持续时长：约 1.0-1.8s（视路径长短）

EF04 — 区域修复光效（Area Restore Glow）
- 触发：区域修复阶段切换时
- 画布：覆盖区域图块的全区域（约 400×400 @2x）
- Spine文件：fx_area_restore.skel，含 4 个阶段动画（stage1-4）
- 内容：白色/金色光波从区域中心向外扩散，持续约 3-5s，配合区域图片crossfade切换
- 第4阶段（完整修复）：额外该章节主题粒子雨（樱花章节散落花瓣；神社章节散出光点；灯塔章节金色光芒），约 2.5s

EF05 — 河豚爆炸特效（Pufferfish Pop）
- 触发：炸弹（河豚）块激活时
- 画布：640×640 @2x（3×3格范围）
- Spine文件：fx_pufferfish_pop.skel
- 内容：中央河豚夸张膨胀（scale 1.0→2.0，0.3s）→ "啵"一声爆开（圆形白色冲击波向外扩散）→ 大量圆形气泡粒子向外飞散（#A8E0D8浅蓝绿色气泡，直径8-24px随机）→ 泡泡逐渐升起消散
- 总时长：约 0.8s（比原炸弹特效更慢更轻盈，保持治愈感）
- 无火焰粒子，气泡替代碎片

EF06 — 金枪鱼闪电特效（Tuna Dash）
- 触发：闪电（金枪鱼）块激活时
- 画布：1080×160 @2x（横向全行）或 160×1920 @2x（纵向全列）
- Spine文件：fx_tuna_dash_row.skel / fx_tuna_dash_col.skel
- 内容：金枪鱼剪影从激活块高速横扫全行（留下银蓝色#7ECDE8光影残像3-5帧）→ 扫过的图块短暂闪白→ 金枪鱼在边缘消失（溅出水花）
- 总时长：约 0.5s（横扫速度 2000px/s）

EF07 — 彩虹鱼爆发特效（Rainbow Fish Burst）
- 触发：彩虹鱼块消除时
- 画布：320×320 @2x
- Spine文件：fx_rainbow_burst.skel
- 内容：彩虹鱼旋转一圈（360°，0.3s）→ 爆开为7条颜色各异的鱼鳞片，向外放射扩散（鳞片形如椭圆，每片携带不同彩色）→ 鳞片淡出消失
- 总时长：约 0.8s

EF08 — 灯塔光柱特效（Lighthouse Beam）
- 触发：第6章，每8步自动触发 / 标志性时刻全岛版
- 画布：全屏 1080×1920 @2x
- Spine文件：fx_lighthouse_beam_row.skel（单行扫过）/ fx_lighthouse_beam_island.skel（全岛扫过）
- 内容：金黄光柱（宽约80px，#F5C842+透明度渐变边缘）从右侧灯塔方向扫过棋盘，光柱内所有Glow萤火块的光芒瞬间加强（亮度+40%）
- 单行版总时长：约 0.6s；全岛标志性时刻版：约 4s（缓慢旋转扫过全图）

7. 交付标准

7.1 通用交付要求

要求项 说明
分辨率 全部以 @2x（高分辨率）为主交付，@1x 可通过自动缩小生成
色彩空间 sRGB
文件大小 单张 PNG ≤ 500KB（大背景 ≤ 1.5MB），Spine 纹理 ≤ 512KB
透明通道 使用透明背景的图，确保边缘无多余像素（无白边/黑边溢出）
命名 全部小写+下划线，含@2x/@1x后缀，不使用中文或空格
版本控制 文件名含版本号时使用 _v1, _v2 后缀
风格一致性检查 所有图块在同一灰度图中，色彩明度排序符合Art Direction第2.4节对比度要求

7.2 Spine 动画交付要求

要求项 说明
Spine 版本 4.1.x（与引擎版本对齐）
导出格式 Binary .skel + JSON .json（备用）+ .atlas + PNG 纹理
纹理尺寸 2 的幂次：256/512/1024
命名规范 Spine 内部动画名称：全英文小写+下划线，如 idle, activate, talk, react_happy
槽位（Slots） 可替换颜色/内容的部分使用 Slot，便于程序化控制（如Combo气泡中的数字文字）
