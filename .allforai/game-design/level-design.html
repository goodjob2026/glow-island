3. 难度曲线

3:1 节奏设计（喘息关）

全游戏采用 3:1 标准:喘息 节奏模型。每3关标准难度后安排1关喘息关（明显降低目标数与步数压力），防止玩家疲劳流失。

标准关 ×3
喘息关 ×1

喘息关特征：目标数较前3关降低约30%，emotional_note 带有轻松意象（如 "Breather: sit on the pier" / "a bamboo ladle of spring water"）。

全局难度分布

全游戏呈6段锯齿上升曲线：每章开头3关难度回落约15%（章节缓冲期），章内按宽松→标准→挑战三段递进。

4. 教学设计

关卡 1-1、1-2、1-3 为无文字教学（zero-text tutorial），全部核心概念通过关卡布局与视觉引导传递，不出现教学弹窗。

1-1：初识连接

设计意图让玩家直觉发现"同色图块可连接"。4×4 最小盘面，仅 T01/T02 两色，目标仅4块。布局预置两条显而易见的同色直线，玩家几乎不会失败。成功消除时同步播放粒子光效，强化操作反馈。
核心传达连接 → 消除 → 得分（无步数压力）
情感基调"First steps on the shore — the sea greets you gently."

1-2：数量目标感知

设计意图目标从4块提升至6块，引入"目标计数器"视觉元素。玩家需要多连接两次才能达成，自然理解"完成目标才能通关"的循环。仍无步数限制，保持治愈节奏。
核心传达目标数量 = 进度感知（UI计数器）
情感基调"Warm sand under your feet, a simple puzzle to start."

1-3：折线路径探索

设计意图预置一组需要折线（L形路径）才能连接的同色图块。玩家若只尝试直线连接会失败，被迫探索折线规则。关卡布局通过颜色引导视线——折线两端用同色高亮图块，减少试错次数。
核心传达连接路径可折线（最多2次转弯）
情感基调"The dock creaks; a good omen for matching."
三关结束后，玩家无需任何文字说明即可掌握核心连连看操作；1-4起可安全引入第三种颜色 T03 与更大目标数。

5. 目标类型演进

目标类型随章节线性堆叠，最终章允许5种目标同时出现，形成最高复杂度的组合挑战。

章节 可用目标类型 新增 典型组合示例
Ch1 clear_tiles clear_tiles 消除任意 N 块
Ch2 Ch1 + clear_ice clear_ice 消除N块 + 解冻M块冰
Ch3 Ch2 + free_chains free_chains 消除N块 + 解冻M冰 + 解开K锁链
Ch4 Ch3 + traverse_path traverse_path 消除 + 解冻 + 解链 + 穿越L条通道
Ch5 Ch4（无新目标，深化Board Events） — 组合目标 × 动态棋盘事件（下落/滑动/旋转）
Ch6 Ch4 + survive_spread survive_spread 消除 + 解冻 + 解链 + 穿越 + 阻止扩散 （全组合）

6. 关卡格式规范

每个关卡以 JSON 对象形式存储于 level-design.json 的 levels 数组中。关卡编辑器导入时需严格符合以下字段规范。

顶层字段

字段名类型必填说明
idstring REQUIRED 唯一关卡标识，格式 "<chapter>-<level>"，如 "1-1"、"6-23"
chapterinteger REQUIRED 所属章节编号，1–6
levelinteger REQUIRED 章内关卡序号，从1开始
grid_size[int, int] REQUIRED 棋盘尺寸 [宽, 高]，有效范围 [4,4]~[10,10]
tile_typesstring[] REQUIRED 本关启用的图块类型列表，如 ["T01","T02","T03"]；需为已解锁章节的图块子集
objectivesObjective[] REQUIRED 胜利条件数组，至少1条，见下方 Objective 结构
max_movesinteger | null REQUIRED 最大步数限制；null 表示无限制（仅用于早期教学关）
obstaclesObstacle[] REQUIRED 障碍对象数组，空数组表示无障碍
special_blocksSpecialBlock[] REQUIRED 预置特殊块数组，空数组表示无
board_eventsBoardEvent[] REQUIRED 棋盘事件数组，空数组表示无动态事件
material_reward{ type, amount } REQUIRED 通关材料奖励，type 须匹配章节材料类型
coin_rewardinteger REQUIRED 基础金币奖励（三星时另有额外奖励，由评分系统计算）
emotional_notestring OPTIONAL 情感注释，供美术/叙事参考，不影响游戏逻辑

Objective 结构

字段类型说明
typestringclear_tiles / clear_ice / free_chains / traverse_path / survive_spread
tile_typestring仅 clear_tiles 使用，值为 "any" 或具体类型如 "T03"
countinteger目标数量；traverse_path 时为 corridor_count
tile_countinteger仅 survive_spread 使用，表示需保护的格子数
spreading_obstacle_typestring仅 survive_spread 使用，值为 "spreading_obstacle"

Obstacle 类型枚举

type 值引入章节关键子字段
ice_blockCh2position, layers (1–3)
weedCh2position
chain_lockCh3position, required_matches (1–3)
portalCh3position, pair_id, role ("entry"/"exit")
single_path_corridorCh4cells (坐标数组), direction
wooden_crateCh4position
water_currentCh5position, initial_direction, change_interval_s
spreading_obstacleCh6position, variant ("moss"/"vine"), spread_interval_moves

type 值引入章节触发条件
