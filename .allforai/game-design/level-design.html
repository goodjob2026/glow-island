棋盘形状演进

随章节推进，棋盘从标准矩形逐步演变为符合輝島地形的异形棋盘，策略空间与视觉叙事同步增长。

Ch1–2：标准矩形（8×8 / 9×9），零认知负担，专注机制学习。
Ch3：前10关矩形过渡，第11关起引入海湾轮廓异形，有效格数保持偶数。
Ch4–5：岛屿轮廓异形，void格子标记后有效格保持连通。
Ch6：灯塔岛轮廓，终章最大扩展至10×10，Boss关满棋盘布局。

3. 特殊图块系统 v4.0

v4.0 完全重设计。旧系统（bomb炸弹 / windmill风车 / lantern灯光块 / 旧wave海浪块）全部废弃，替换为5种战术型棋盘特殊图块。所有新特殊图块由设计层面保证奇偶安全，无需运行时自动修正。

cascade 连锁图块 第一章起
生成方式Combo=3时在最近消除位自动生成；关卡编辑器可预置
触发玩家将两个连锁图块配对连线激活
效果引擎自动执行最多5次合法配对（0.3秒间隔展示），每次自动配对各计入Combo链
奇偶安全仅执行路径可连通的合法配对，始终偶数消除
light_chain 光链图块 第二章起
生成方式Combo=4时在最近消除位自动生成；关卡编辑器可预置
触发玩家将两个光链图块配对连线激活
效果全图扫描数量最多的图块类型，自动匹配所有可组成的配对并链消；余1块不消除
奇偶安全只消除完整配对，奇数个时留最后1块，天然安全
wave 光波图块 第三章起
生成方式Combo=6时生成（重磅奖励）；关卡编辑器可预置
触发玩家将两个光波图块配对连线激活
效果清除以连线中点为中心的3×3区域内所有障碍层（不消除匹配图块），并赋予该区域5秒自由路径增益
奇偶安全障碍不是配对图块，清除障碍完全不影响任何图块奇偶，零风险
pierce 穿透图块 第四章起（仅关卡预置）
生成方式仅关卡编辑器预置（战略性谜题元素，不由Combo自动生成）
触发玩家将两个穿透图块配对连线激活
效果赋予『穿透模式』8秒增益：连线路径可穿越棋盘上任意已占据格子（障碍或图块均可穿越），同类图块匹配规则不变
奇偶安全不移除任何图块，仅修改路径路由规则，奇偶完全不受影响
swap 置换图块 第五章起（仅关卡预置）
生成方式仅关卡编辑器预置（高技巧战术元素）
触发玩家将两个置换图块配对连线激活
效果玩家进入『选择模式』10秒，在棋盘上选择任意两格，其内容（图块/障碍/空格）互换位置；10秒未选择则置换图块保留等待再次配对
奇偶安全只交换位置，不增减任何图块，总计数不变，奇偶完全不受影响
旧版特殊图块已废弃：bomb（炸弹）、windmill（风车）、lantern（灯光块）、旧版wave（海浪块行滑动）均不再使用。
新版5种图块均不消除单格匹配图块，奇偶安全由设计层面保证，无需运行时自动修正逻辑。

4. 难度曲线

首次通关率目标与降级机制

设计目标首次通关率 ≥ 70%（首次游玩通关关卡的玩家比例）
降级触发通关率低于 50% 时触发关卡难度降级建议（减少障碍或增加步数）
步数续关步数耗尽后可花费30沙滩币购买+5步续关，降低硬卡关流失

3:1 节奏设计

全游戏采用 3:1 标准:喘息 节奏模型。每3关标准难度后安排1关喘息关，防止玩家疲劳流失。

标准关 ×3
喘息关 ×1

喘息关特征：目标数较前3关降低约30%，emotional_note 带有轻松意象；步数余量充裕。

禅境关（每章2关）：完全关闭步数上限和失败判定，零压力连击体验。

全局难度分布（24个难度段）

全游戏呈6段锯齿上升曲线：每章开头难度回落约15%（章节缓冲期），章内按宽松→标准→挑战三段递进，Boss关为当章最高峰值。

8. 关卡格式规范 v3.0

每个关卡以 JSON 对象形式存储，供关卡编辑器导入。v3.0 规范与 core-mechanics v4.0 完整对齐。

顶层字段说明

字段名类型必填性说明
idstring必填唯一关卡标识，格式 "<章节>-<关卡序号>"，如 "1-1"、"6-30"
chapterinteger必填所属章节编号，1–6
levelinteger必填章内关卡序号，1–30
level_rolestring必填关卡角色：tutorial / standard / story / boss / zen / breather
grid_size[int, int]必填棋盘尺寸 [宽, 高]，有效范围 [8,8]~[10,10]
shapestring必填棋盘形状：rectangle / bay_contour / island_contour / lighthouse_contour
tile_typesstring[]必填本关启用的图块类型列表，如 ["T01","T02","T03"]；须为已解锁章节的图块子集
objectivesObjective[]必填胜利条件数组，至少1条，见下方 Objective 结构说明
max_movesinteger | null必填最大步数限制；null 表示无限制（禅境关与早期教学关专用）
obstaclesObstacle[]必填障碍对象数组，无障碍时为空数组 []
special_tilesSpecialTile[]必填预置特殊图块数组（cascade/light_chain/wave/pierce/swap），无时为 []
board_eventsBoardEvent[]必填棋盘事件数组，无动态事件时为 []
material_reward{ type, amount }必填通关材料奖励，type 须匹配本章材料类型
coin_rewardinteger必填通关基础沙滩币奖励
emotional_notestring可选情感注释，供美术/叙事参考，不影响游戏逻辑；文字用中文
narrative_triggerstring可选叙事触发类型：over_scene / npc_dialogue / diary_fragment
teaching_beatstring可选关联教学节拍ID，如 "tb-001"
zen_badgestring可选禅境关装饰性徽章名称（仅 level_role=zen 时有效）

Objective 结构说明

字段类型说明
typestringclear_tiles / clear_ice / free_chains / traverse_path / survive_spread
tile_typestring仅 clear_tiles 使用，值为 "any" 或具体类型如 "T03"
countinteger目标数量；traverse_path 时表示 corridor_count（通道数）
tile_countinteger仅 survive_spread 使用，表示需保护的有效格数量上限
spreading_obstacle_typestring仅 survive_spread 使用，值为 "spreading_obstacle"

障碍类型枚举（按章节引入顺序）

type 值引入章节关键子字段
ice_block第二章position, layers (1–3)
weed第二章position
chain_lock第三章position, required_matches (1–3)
portal第三章position, pair_id, role ("entry"/"exit")
single_path_corridor第四章cells（坐标数组）, direction
wooden_crate第四章position（永久阻挡格）
water_current第五章position, initial_direction, change_interval_s
spreading_obstacle第六章position, variant ("moss"/"vine"), spread_interval_moves

特殊图块枚举 v4.0（已废弃旧版本）

type 值引入章节生成方式
cascade第一章Combo=3自动生成 / 关卡预置
light_chain第二章Combo=4自动生成 / 关卡预置
wave第三章Combo=6自动生成 / 关卡预置
pierce第四章仅关卡预置
swap第五章仅关卡预置

⚠ 已废弃：bomb / windmill / lantern / 旧版wave。请勿在 v3.0 关卡文件中使用。

棋盘事件枚举

type 值引入章节关键子字段
tile_fall第五章enabled: true
tile_slide第五章enabled, direction
board_rotate第五章（仅挑战关）enabled, interval_moves
vine_spread第六章enabled, spread_interval_moves
