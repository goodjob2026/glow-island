组件状态规格

以下为所有 UI 组件完整状态机，满足 Cocos Creator 2D UI 实现需求。

重排/预判按钮（icon_button 共通状态）

normal 珊瑚渐变，白色图标，white 标签，soft shadow 默认状态 & 余额 ≥ 30
pressed 加深珊瑚色，Y轴+2px，scale 0.97，阴影减半 pointer_down
disabled 沙色 #C8B8A0，图标/标签灰，opacity 0.6，无阴影 coin_balance < 30
locked disabled + 锁图标叠加，opacity 0.5 无合法目标（棋盘已清空等）

续关按钮（额外 highlighted 态）

normal 沙色调，opacity 0.7（低亮度常驻） 步数 > 0
highlighted 完整珊瑚渐变，金色边框 #F5C842，脉动光晕，opacity 1.0 step_count == 0 AND balance ≥ 30
disabled 灰色 #C8B8A0，opacity 0.5 coin_balance < 30
locked dim opacity 0.3，锁图标 禅境关（zen_mode）或设计限制

步数计数器告警颜色

状态触发条件颜色动效
normal步数 > 10#3A2810无
low步数 ≤ 10#EE8C55gentle_pulse 0.8s
critical步数 ≤ 5#C85A40fast_pulse 0.4s
empty步数 = 0#C85A40续关按钮高亮激活

灯塔进度条状态

状态触发条件填充色动效
empty / critical进度 0%（修复前）#C8B8A0（灰）灯塔图标暗淡无
low进度 < 25%#C4B090无
normal进度 25%–99%#7DBE77（竹绿）smooth expand + 金色闪光 0.3s（gain 时）
full进度 100%#F5C842（金黄）灯塔图标发光golden_glow_pulse 1.0s 循环

审查清单（review_checklist）

  • ✓
    UI 组件状态完整（normal/pressed/disabled/locked）

    重排/预判/续关按钮均有 4–5 态完整定义；HUD 底栏有 normal/low/critical/empty 4态；对话框和商店面板有 entering/open/confirming/closing/blocked 5态。详见 component-state-spec.json。

  • ✓
    HUD 信息层级清晰，不遮挡关键棋盘区域（z-index 正确）

    HUD 底栏（y=1100–1180）与棋盘（y=100–1100）无重叠。灯塔侧栏（x=660–720）不遮挡主棋盘（x=0–660）。z-index：背景0→棋盘1→VFX2→HUD3→弹窗10。详见 screen-layout-spec.json。

  • ✓
    屏幕布局规格与 Cocos Creator 分辨率适配（9:16基准+安全区）

    基准分辨率 720×1280（9:16），顶部安全区 44px（刘海），底部安全区 34px（Home 条）。screen-layout-spec.json 中所有 breakpoint 已按此规格定义。

  • ✓
    图标与整体美术方向一致，Atlas 打包无越界

    所有 UI 资产使用 hand-drawn watercolor + warm pastel 风格，禁止像素艺术/霓虹/金属铬渐变。全部10个 UI 资产打包至单一 ui_elements_atlas（max 2048px），已在 atlas_plan 中定义。

  • ○
    UI mockup 通过可读性 QA

    待执行：图片生成后在 art-qa 节点执行可读性验证。当前所有 mockup 为 spec_ready / not_generated 状态。提示词已定义（见 ui-mockup-spec.json），包含文字适配检查和布局保真度检查。
