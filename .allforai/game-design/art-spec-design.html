饱和度修复分阶规则

同一图块在 6 个章节图集中使用不同饱和度参数，视觉上体现輝島的修复进度。

修复阶段饱和度范围亮度视觉描述应用章节
s0（修复前）8–15%42–48%褪色枯岛，铅笔草稿感Ch1 初始状态
s128–38%–轮廓找回，淡彩涂色Ch1 修复后
s248–58%–生活气息，色彩入稿Ch2–Ch3
s362–72%–角色回归，满生命力Ch4–Ch5

饱和度过渡方式：crossfade 0.8s 每阶段切换。Ch6 灯塔光柱扫过时饱和度瞬间提升至 100% 再恢复。

眨眼频率每 5 秒一次，持续 0.15 秒
随机偶发动作每 8–12 秒：左右摇头 ±5° / 手臂摆动 ±8° / 短暂抬头
头身比例1:1.5 Q版；头部正圆形占全身高 40%

身形身体圆筒形无腰线；四肢短圆；脚部略大于手（增加稳定感）
剪影策略每位 NPC 单色剪影，任意分辨率下可识别
格式PNG 精灵图 + DragonBones JSON 骨骼数据
图集characters_atlas（统一）
AI 生成否（需人工制作骨骼绑定与表情调整）

5 种特殊图块激活效果说明

特殊图块名激活动效描述持续时长主色

光链（light_chain） 金色光环脉冲两下 → 扫描波横扫棋盘 → 目标类型所有图块同时高亮金色 → 连线同时射出，链式消失，节奏递进 0.8s #F5C842
穿透（pierce） 蓝白光箭爆发 → 屏幕四角持续蓝色边缘光晕（增益 8s 存续标志）→ 增益期间连线呈发光光束 → 8s 后渐退 激活 0.3s + 8s 增益期 #7ECDE8
置换（swap） 两个绿色旋转螺旋出现在被选中格子 → 相向旋转加速 → 图块互换弹性『咔哒』特效 → 绿色光晕渐退；10s 计时器显示于棋盘角落 选择计时 10s，交换动画 0.4s #4A7D55
连锁（cascade） 橙红波纹向外扩展 → 聚光灯锁定第一对配对 → 自动连线消除 → 0.3s 延迟后锁定下一对 → 循环往复，节奏逐渐加快，音效强度递增 每对 0.3s 延迟，最多 5 对 #C85A40

图集策略设计决策

为什么按章节分图集？

Web-canvas-2d 平台需严格控制内存。按章节图集允许按需加载，玩家推进时才载入下一章图集，保持活跃 VRAM 占用最小化。

为什么特殊图块独立图集？

所有 5 种特殊图块可在任意章节出现（连锁从 Ch1 起即可用），其图块图标不绑定章节颜色变体，因此需独立于章节图集存在，避免重复打包。

分辨率降级策略

所有资产以 2x 分辨率设计，运行时根据设备 DPR 自动降级至 1x。Cocos Creator 自动图集打包工具统一处理，不需要手动维护两套资产。

VFX 图集单独限制为 1024 px

VFX 帧序列本身在 GPU 上为临时纹理，使用频率高、更新快，限制为 1024px 图集可在移动端浏览器保持稳定帧率。

禁用风格（出现即驳回）

  • 像素艺术 (pixel art)
  • 写实3D拟物化 (hyper-realistic 3D)
  • 霓虹 / 赛博配色（#00FFFF 类纯荧光色）
  • 金属铬钢高光渐变
  • 纯黑色 #000000（必须替换为深棕 #3A2810 或深蓝 #1E2E50）
  • 锐利直角几何 UI 元素
  • Material Icon 风格锐利矢量图标
  • 过度粒子特效（屏幕碎片满天飞超过预算）
  • 强对比警示色调（破坏治愈调性）

色彩对比规则（全部必须满足）

规则最低要求
相邻章节图块色相差≥ 30°（确保章节视觉可区分）
同章节内图块明度差≥ 20%（避免图块视觉混淆）
图块最低饱和度≥ 20%（禁止纯灰色图块）
背景装饰透明度≤ 15%（不与图块色相重叠干扰）
灰度模式可识别性任意两种图块在灰度模式下仍可区分（辅助功能要求）
